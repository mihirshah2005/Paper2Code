4. Dirty Sample Injection (inject_dirty_samples Method)

This method is central to simulating the experimental conditions and involves two independent injections:
 A. Noisy Label Injection  
 B. Backdoor (Poisoned) Sample Injection

A. Noisy Label Injection:
 – Using the noisy ratio provided (η = 0.4 in the configuration), iterate over each sample in the training set.
 – For each sample, with probability η, modify the sample’s label.  
  • For symmetric noise, the original label is replaced by randomly selecting one of the remaining (K – 1) labels.
  • For asymmetric noise, the label is modified by following a predetermined rule such as replacing it with the “next” label; mathematically, (i mod K) + 1.
 – This process simulates annotation errors and is applied to the dataset as a whole.

B. Backdoor (Poisoned) Sample Injection:
 – Based on the poisoning configuration for the given dataset, randomly select a specified number of samples per class (e.g., 50 or 500 for CIFAR-10, 5 or 50 for ImageNet-100, 80 for ImageNet-Dog).
 – For each selected sample (typically taken from non-target classes), a backdoor trigger is injected by overlaying a predefined pattern.  
  • Examples:  
   – For BadNets, a 3×3 white square is inserted in the lower-right corner (CIFAR-10), scaled appropriately for ImageNet images.  
   – For Blended attacks, a “Hello Kitty” image is blended with the original image with a ratio of 0.1.  
   – For SIG and other attacks (TrojanNN, SSBA, WaNet), follow the specific parameters provided, with the overlay details delegated to utility functions.
 – After the visual trigger is applied, the sample’s label is changed to the designated target label (commonly fixed as class 0).

C. Injection Order and Overlap Clarification:
 – The default design applies the two injection schemes sequentially on the same training dataset:
  1. First, the noisy label injection is applied to all samples based on the noisy ratio.
  2. Then, the backdoor injection is performed by selecting samples from the (already modified) dataset.
 – With this sequential order, overlap is permitted; that is, a sample that has already received a noisy label may also be chosen for backdoor injection. This results in hybrid dirty samples—an experimental setup that reflects scenarios where datasets contain both types of errors.
 – Alternatively, an optional configuration flag may be introduced in the future to enforce mutual exclusivity (i.e., to prevent overlapping modifications). By default, however, the design follows the sequential approach, allowing both injections to occur on the same sample.
 – This detailed order is documented in the code comments to avoid any ambiguity for future maintainers and ensure that the injection process is fully reproducible.

After processing through both steps, the modified training dataset (which now may contain clean, noisy, poisoned, and hybrid dirty samples) is returned. Detailed logging (using helper functions from utils) is performed to record the number of samples modified by each injection, which later assists in computing metrics such as TPR and FPR.
