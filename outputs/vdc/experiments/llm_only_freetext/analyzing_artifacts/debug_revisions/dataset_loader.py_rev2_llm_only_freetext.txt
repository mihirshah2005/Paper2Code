4. Dirty Sample Injection (inject_dirty_samples Method)

This method is central to simulating the dirty samples according to the paper. It handles two sub-tasks:
 A. Noisy Label Injection  
 B. Backdoor (Poisoned) Sample Injection

A. Noisy Label Injection:
 – The configuration provides a noisy ratio (η = 0.4). For each training sample, with probability η, the sample’s label is modified.
 – Two strategies are available:
  • Symmetric noise: Randomly replace the true label with one of the other (K – 1) labels.
  • Asymmetric noise: Replace the true label with a “next class” label (using modulo arithmetic: (i mod K) + 1).
 – This injection is applied across the dataset and is designed to mimic annotation errors.

B. Backdoor (Poisoned) Sample Injection:
 – For each dataset, the configuration specifies poisoning parameters—for example:
  • CIFAR-10: Choices include injecting 50 or 500 poisoned samples per class.
  • ImageNet-100: 5 or 50 poisoned samples per class.
  • ImageNet-Dog: 80 samples per class.
 – The process involves:
  • For each (non-target) class, randomly selecting the designated number of samples.
  • For each selected sample, overlay a trigger image according to the selected attack (e.g., BadNets using a 3×3 white square trigger, Blended using a “Hello Kitty” image blended at a ratio of 0.1, SIG with the defined sinusoidal parameters, etc.).  
  • The actual overlay mechanics are delegated to utility functions (in utils.py) to enforce consistency.
  • After injecting the visual trigger, update the sample’s label to the defined target label (typically class 0).

C. Injection Order and Overlap Handling:
 – This revised analysis explicitly clarifies that the two injection processes (noisy label and poisoned sample injection) are applied in a sequential order.
 – DESIGN DECISION:  
  Option 1 – Allow Overlap (Hybrid Dirty Samples):  
   • Both injections are applied independently. That is, a sample might first receive a noisy label modification and then, if randomly selected for poisoning, also have a trigger overlay applied. This results in a hybrid dirty sample, which is consistent with the paper’s hybrid dirty sample scenario.
  Option 2 – Enforce Mutual Exclusivity:  
   • Alternatively, the implementation can be configured (via an additional parameter) such that if a sample is chosen for poisoning, it is excluded from noisy label injection, or vice versa.  
  • By default, to fully reproduce the hybrid dirty sample setting described in the paper, it is preferable to allow overlap: the injection functions should operate on the full dataset independently, resulting in some samples being modified by both procedures.
 – The inject_dirty_samples method should document this behavior clearly and log which samples have been modified, so that downstream evaluation (e.g., computation of TPR and FPR) can distinguish hybrids if necessary.
 – All random selections must use fixed random seeds (using utils functions) to ensure reproducibility.

After applying both injections (in the chosen sequential order), the method returns the modified/training dataset.
