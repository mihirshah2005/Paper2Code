The load_data() function will handle three main cases:
 – For CIFAR-10:
  • Use torchvision.datasets.CIFAR10 with train=True and train=False.
  • Provide the appropriate transform pipeline for training and testing.
  • No custom filtering is necessary since the dataset is standard.
 – For ImageNet-100:
  • Although torchvision may not have a pre-specified “ImageNet-100”, the loader will use an ImageFolder-based loader.
  • A helper function (or inline filtering logic) will be used to randomly select 100 classes from ImageNet.
  • Ensure that for training, only 500 images per class are kept and for testing only 100 images per class are retained.
  • The transforms will be defined for ImageNet images (e.g., resize to 224×224, center crop, normalization with ImageNet means).
 – For ImageNet-Dog:
  • Use a similar ImageFolder-based loader and filter the dataset to only include 10 dog classes.
  • Make sure that the selected dog datasets conform to the paper’s specification of having 800 images per class for training and 200 images per class for testing.
 – Finally, return a dictionary (or similar structure) containing “train_data” and “test_data” for the selected dataset.

4. Dirty Sample Injection (inject_dirty_samples Method)

This method takes the originally loaded training data (e.g., a list or a PyTorch Dataset) and modifies it to simulate “dirty” samples. The injection process includes two parts:

A. Noisy Label Injection:
 – According to the config, a noisy ratio (η = 0.4) is applied.
 – For each sample in the training set, with probability η, its true label can be replaced by:
  • For symmetric noise: a random label selected uniformly at random from the set of labels excluding the true label.
  • For asymmetric noise: change the label according to a rule (for example, (i mod K) + 1, where i is the original label index and K is the total number of classes).
 – The implementation should be careful to work on clean copies of data (or modify in place with reproducible randomness).

B. Backdoor (Poisoned) Sample Injection:
 – Retrieve the poisoning settings from the config for the current dataset. For example:
  • CIFAR-10: two possibilities: 50 and 500 poisoned samples per class.
  • ImageNet-100: 5 and 50 polluted samples per class.
  • ImageNet-Dog: 80 samples per class.
 – Follow the paper’s procedure: for each class (except the target class, which is fixed to class 0 per the paper’s example), randomly choose the specified number of samples.
 – For each selected sample:
  • Overlay a trigger using one of the backdoor attack functions (e.g., BadNets, Blended, SIG, etc.). The actual overlay implementation is delegated to utility functions provided by utils.py.
  • Change the label of the modified sample to the target label (usually class 0).
 – Depending on the implementation of multiple backdoor attack schemes, you might allow choosing one (or multiple) methods; however, at minimum the simple ones (e.g., BadNets or Blended) should be implemented.
 – It is important to update both image data (with the trigger overlaid) and labels (changed to target).

• The inject_dirty_samples method should then return the updated training data structure. Logging (or print statements via a helper utility) can be used to report the number of noisy and poisoned samples injected.
