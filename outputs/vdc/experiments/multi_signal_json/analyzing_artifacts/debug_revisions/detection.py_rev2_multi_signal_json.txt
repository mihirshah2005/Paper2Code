1. Initialization

• The VDCDetector class is instantiated with module instances:
  – VQG: Generates both general questions (using fixed templates, number defined by config) and label-specific questions (using ChatGPT API with provided prompts).
  – VQA: Answers each question for a given image using Instruct-BLIP (or a similar multimodal model).
  – VAE: Evaluates the answers:
    ▪ For label-specific questions: Uses straightforward string matching.
    ▪ For general questions: Invokes ChatGPT via an evaluation prompt to assess answer correctness.
• The detector also stores a threshold value (e.g., 0.5) read from the configuration file. This threshold is used to determine if the overall matching score indicates that the sample is “dirty.”

2. Detection Process in detect_sample

The detect_sample method processes a sample (image paired with label) according to the following steps:

Step 1: Question Generation  
  • Call the VQG module using two methods:
    – generate_general_questions(label)
    – generate_label_specific_questions(label)
  • Both calls are wrapped in try/except blocks to catch API or network errors. For example, if an API call fails, a fallback behavior or default set of questions (from a pre-cached version) can be used.
  • Tag each generated question with its type ("general" or "label_specific") and combine the two lists.

Step 2: Answer Retrieval  
  • For each question in the combined list:
    – Invoke VQA.answer_question(image, question) to receive an answer.
    – Wrap this call in a try/except block; if the answer retrieval fails (e.g., due to an API timeout), log the error and optionally assign a default answer (or mark that question as unanswered).
  • This ensures that an API failure in the VQA module does not break the detection pipeline entirely and that a fallback is provided.

Step 3: Answer Evaluation  
  • For each produced answer, use VAE.evaluate_answer(answer, expected_answer, question_type) to decide if the answer is “correct”:
    – For label-specific questions, the expected answer is “yes” (after normalization).
    – For general questions, the expected answer is determined internally by the evaluation prompt.
  • The VAE call is also enclosed within a try/except block. In the event of an evaluation API failure, the module might return a default negative evaluation (or retry based on configuration), and the issue is logged.
  
Step 4: Compute the Matching Score  
  • Count the number of questions with a “correct” evaluation (either successful – yes – or a default result if a failure occurred and was handled).
  • Compute the matching score sᵢ = (number of successful evaluations) / (total number of questions generated).

Step 5: Decision Making  
  • Compare sᵢ to the threshold (α = 0.5):
    – If sᵢ ≥ 0.5, the sample is considered clean.
    – Else, mark it as dirty.
  • Return the Boolean decision accordingly.

4. Pseudocode Outline with Exception Handling

Below is the updated pseudocode that includes explicit exception handling:

  def detect_sample(self, image, label) -> bool:
      try:
          general_questions = self.vqg.generate_general_questions(label)
      except Exception as e:
          log("Error in generating general questions:", e)
          general_questions = self.vqg.get_cached_general_questions(label)
      
      try:
          label_specific_questions = self.vqg.generate_label_specific_questions(label)
      except Exception as e:
          log("Error in generating label-specific questions:", e)
          label_specific_questions = self.vqg.get_cached_label_specific_questions(label)
      
      all_questions = [(q, "general") for q in general_questions] + [(q, "label_specific") for q in label_specific_questions]
      
      total = len(all_questions)
      correct_count = 0
      
      for question, q_type in all_questions:
          try:
              answer = self.vqa.answer_question(image, question)
          except Exception as e:
              log("Error in VQA answer_question:", e)
              answer = ""  # fallback, could mark as incorrect automatically
          
          expected_answer = "yes" if q_type == "label_specific" else None
          
          try:
              evaluation_correct = self.vae.evaluate_answer(answer, expected_answer, q_type)
          except Exception as e:
              log("Error in VAE evaluate_answer:", e)
              evaluation_correct = False
          
          if evaluation_correct:
              correct_count += 1
      
      matching_score = correct_count / total if total > 0 else 0
      
      return matching_score >= self.threshold
