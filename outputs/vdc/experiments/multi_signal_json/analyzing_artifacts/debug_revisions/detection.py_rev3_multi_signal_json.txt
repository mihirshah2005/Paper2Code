2. Detection Process in detect_sample

The detect_sample method follows these steps:

Step 1: Question Generation  
  • Call the VQG module’s methods for question generation:
    – For general questions: Call generate_general_questions(label).  
    – For label-specific questions: Call generate_label_specific_questions(label).  
  • Each call is wrapped in a try/except block.  
    – If an exception occurs (for example, if a ChatGPT API call fails), the VQG module is expected to return a built‐in default list of pre-defined questions (stored internally by the VQG object).  
    – This default list is defined and maintained as part of the VQG module’s implementation, so the design does not rely on externally defined methods like get_cached_general_questions.
  • Each generated question is tagged with its type (“general” or “label_specific”) and both lists are merged into one overall set.

Step 2: Answer Retrieval  
  • For each generated question in the merged set, call the VQA module:
    – Use VQA.answer_question(image, question) to obtain an answer string.
  • Crucially, if the VQA call fails (e.g., due to an API exception), the exception is caught.  
    – Instead of simply assigning an empty string (which might be misinterpreted by the VAE module), the design now directly marks that particular question as having failed—equivalently returning a “negative” evaluation.  
    – In practice, this means that if answer retrieval fails for a question, the code will not attempt to evaluate an empty answer via VAE, but will instead count that question as incorrectly answered.  
    – This ensures that a retrieval failure will properly lower the sample’s overall matching score without introducing a bias that might occur if an empty string were later (unexpectedly) interpreted as correct.

Step 3: Answer Evaluation  
  • For each answer generated (or for questions marked as failed), the VAE module is invoked to decide if the answer is semantically correct.  
    – For label-specific questions, the expected answer is typically “yes” (using normalized text comparisons).  
    – For general questions, the evaluation is performed by sending an evaluation prompt to ChatGPT that compares the answer against the expected semantic content.  
  • Each call to VAE.evaluate_answer is wrapped in a try/except block to catch any API-related failures.  
    – On an evaluation failure, the design will mark the question as “incorrect” so as not to overestimate the sample’s quality.
    
Step 4: Compute Matching Score  
  • After processing all question-answer pairs, compute the matching score as:
  
        matching_score = (number of questions evaluated as “correct”) / (total number of questions)
  
  • The use of a vote-based ensemble ensures that each question has equal weight in the final score.
  
Step 5: Decision Making  
  • Compare the computed matching score with the threshold (α = 0.5):
    – If matching_score ≥ α, then the sample is considered clean.
    – Otherwise, it is marked as dirty.
  • Return this Boolean decision so that downstream modules (e.g., for dataset purification) can accurately remove dirty samples.

3. Revised Pseudocode Outline with Robust Fallbacks

Below is the updated pseudocode that incorporates these improvements:

  def detect_sample(self, image, label) -> bool:
      # Generate questions using VQG
      try:
          general_questions = self.vqg.generate_general_questions(label)
      except Exception as e:
          # Log error and use built-in default list of general questions from VQG module
          log("Error generating general questions:", e)
          general_questions = self.vqg.default_general_questions  
      
      try:
          label_specific_questions = self.vqg.generate_label_specific_questions(label)
      except Exception as e:
          # Log error and use built-in default list of label-specific questions
          log("Error generating label-specific questions:", e)
          label_specific_questions = self.vqg.default_label_specific_questions
          
      # Combine and tag questions
      all_questions = [(q, "general") for q in general_questions] + [(q, "label_specific") for q in label_specific_questions]
      
      total = len(all_questions)
      correct_count = 0
      
      # Process each question-answer pair
      for question, q_type in all_questions:
          try:
              answer = self.vqa.answer_question(image, question)
          except Exception as e:
              log("Error during VQA answer_question; marking question as failed:", e)
              # Instead of using an empty string, mark the evaluation directly as incorrect.
              answer = None  # or set a flag, so that evaluation immediately returns False.
          
          # Determine expected answer for label-specific questions; for general questions we use evaluation prompt
          expected_answer = "yes" if q_type == "label_specific" else None
          
          try:
              # If answer is None due to an exception, consider its evaluation as False.
              if answer is None:
                  evaluation_correct = False
              else:
                  evaluation_correct = self.vae.evaluate_answer(answer, expected_answer, q_type)
          except Exception as e:
              log("Error during VAE evaluate_answer; marking evaluation as incorrect:", e)
              evaluation_correct = False
          
          if evaluation_correct:
              correct_count += 1
      
      matching_score = correct_count / total if total > 0 else 0
      
      # Make final decision based on matching score
      return matching_score >= self.threshold
