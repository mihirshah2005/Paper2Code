2. TRAINING LOOP – train() Method

• The train() method is responsible for the main training loop, iterating over a number of epochs as configured.

• For each epoch:
  - The model is set to training mode via model.train().
  - Epoch-level accumulators (for loss, correct predictions, and total samples) are reset.
  - It iterates over batches provided by the DataLoader. For each batch:
       • Input images and corresponding labels are loaded and transferred to the appropriate device.
       • The optimizer’s gradients are zeroed.
       • A forward pass is performed, outputs are obtained, and the loss is computed using the CrossEntropyLoss.
       • Backpropagation is performed (loss.backward()) and the optimizer updates the model parameters with optimizer.step().
       • Predictions are computed (using argmax on model outputs) and used to update the running total of correct classifications.
  - After processing all batches, average epoch loss and training accuracy are computed and logged.

• Scheduler Update Clarification:
  - There is an ambiguity to resolve regarding when scheduler.step() is called. Two approaches are possible:
       1. Call scheduler.step() at the beginning of each epoch so that if the current epoch matches a milestone, the new learning rate is applied immediately for that epoch.
       2. Call scheduler.step() at the end of each epoch so that the learning rate update takes effect in the subsequent epoch.
  - To avoid an off‑by‑one error and ensure full adherence to the decay schedule from config.yaml, the implementation must clarify the intended behavior. For instance, if the configuration’s milestones are designed to apply decay for the current epoch, scheduler.step() should be called at the beginning of the epoch. Otherwise, if the intention is to decay after completing an epoch, the call should occur at the end.
  - The analysis recommends explicitly documenting the choice and ensuring consistency with how the milestones are specified.

• After each epoch:
  - Log and record the average training loss and accuracy.
  - Update the learning rate scheduler (with the chosen call placement for scheduler.step()).
  - Optionally, if a separate validation set exists, perform a validation pass here.

• After completing all epochs:
  - Save the final model state (e.g., using model.state_dict()).
  - Return or log the overall training metrics for later use by the Evaluation module, which will compute ACC and provide the trained model to assess ASR.
