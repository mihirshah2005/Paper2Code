2. TRAINING LOOP: train() Method

Inside the train() method, the steps are as follows:

• Set the model to training mode (model.train()).

• For each epoch (from 0 to epochs-1):
  - Reset epoch-level accumulators for running loss, correct predictions count, and total samples processed.
  
  - Iterate over batches from the DataLoader:
      • For each batch, retrieve input images and corresponding labels.
      • Move the images and labels to the designated device.
      • Zero the gradients in the optimizer.
      • Perform a forward pass on the model to obtain the output predictions.
      • Compute the loss using the CrossEntropyLoss criterion.
      • Perform backpropagation: call loss.backward(), and then optimizer.step() to update the model parameters.
      • Compute the number of correct predictions:
          - For each output, take the index of the maximum predicted value.
          - Compare predictions with the ground-truth labels.
          - Update the epoch accumulators (total samples and count of correct predictions).
      • Optionally, log batch-level loss if fine-grained logging is desired.

  - After processing all batches in the epoch:
      • Compute and log the average training loss and training accuracy for the epoch.
      • Optionally, record the current learning rate (which may change after scheduler.step()).
  
  - Update the learning rate scheduler (scheduler.step()). This reduces the learning rate at the defined milestone epochs as given by the “lr_decay_schedule” from the configuration (e.g., drops at epoch 50 and 75 for CIFAR‑10).

• Optionally perform a validation pass if a validation dataset is available. Although the main evaluation (including ASR computation) is likely handled by a separate Evaluation class, the trainer might log validation accuracy.

• At the end of training:
  - Save the final model state (e.g., model.state_dict()) for later evaluation.
  - Return or log the training history, including the epochs, loss, and accuracy metrics.
