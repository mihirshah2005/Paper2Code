4. Trigger Overlay Functions (Backdoor Attack Implementations)

• Purpose:
  – Implement several functions to inject different types of backdoor triggers into images. These functions simulate the dirty sample generation mechanisms as described in the paper.

• Key Attacks and Their Implementation Strategies:
  
  ▪ BadNets:
    – Function (e.g., apply_badnets_trigger) that inserts a white square patch at a fixed position.
    – For CIFAR-10: a 3×3 white square patch placed in the lower-right corner.
    – For ImageNet-100 / ImageNet-Dog: scale the patch appropriately (e.g., a 21×21 patch).
  
  ▪ Blended:
    – Function (e.g., apply_blended_trigger) that overlays a “Hello Kitty” trigger image onto the source image.
    – Blending should use a ratio of 0.1. The function should blend the trigger with the source image pixel values.
  
  ▪ SIG (Sinusoidal Trigger):
    – Function (e.g., apply_sig_trigger) that generates and overlays a sinusoidal signal onto the image.
    – Use the parameters: ∆ = 20, frequency f = 6.
    – The sinusoidal pattern (computed along horizontal coordinates) should be added to all channels.
  
  ▪ (Optional) TrojanNN, SSBA, WaNet:
    – While these attacks are mentioned in the paper, details might be simplified in our implementation.
    – For TrojanNN: overlay a predefined Apple logo in a fixed region.
    – For SSBA and WaNet: if implementing a full encoder–decoder based generation is too heavy, consider a simplified simulation (or just stub functions with a log message).
    
• Implementation Details:
  – Each trigger function will take a PIL Image (or tensor) as input, apply the transformation, and return the modified image.
  – Use configuration parameters (if available) to adjust parameters (e.g., patch size might depend on the dataset type, which can be inferred from config.yaml).
  
• Key Points:
  – Ensure that trigger injection does not alter the overall color space or dynamic range of the image beyond what is expected.
  – The functions should be deterministic (given a fixed seed) to support reproducibility.
