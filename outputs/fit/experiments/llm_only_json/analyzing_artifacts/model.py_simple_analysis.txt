2. PRETRAINED VAE LOADING

• The initialization process in __init__ will check the configuration (from config.yaml) for the pretrained VAE path.
 – The __init__ method will internally call the load_pretrained_vae method so that the VAE is loaded automatically.
 – The load_pretrained_vae method is provided as a public override only if the user wishes to reload or change the VAE later; in regular use, it is invoked during initialization.
 – This clear separation avoids any ambiguity, ensuring a single point of responsibility while still offering flexibility.

3. LATENT TOKENIZATION & PATCHIFICATION

• After an input image is encoded by the pretrained VAE encoder, the resulting latent representation (typically of shape [B, C, H, W]) is processed via a patchification routine.
 – With a patch size of p = 2, the latent map is divided into non-overlapping patches and then flattened to form a sequence of tokens.
 – Owing to varying image resolutions, each sample yields a different token sequence length. An internal padding mechanism is applied to extend all token sequences to a fixed maximum length (Lₘₐₓ = 256).
 – A validity mask is generated to indicate which tokens are “real” versus padded. This mask is critical both for proper attention masking in the transformer and for computing the diffusion loss only over valid tokens.

────────────────────────────────────────────
4. CORRECT INTEGRATION OF 2D ROTARY POSITIONAL EMBEDDING (RoPE)

• In contrast to simply adding or concatenating positional embeddings, the FiT model employs 2D RoPE by directly modifying the query and key vectors within the attention module:
 – For each token, its spatial coordinates (obtained from the patchified latent grid) are used to compute rotation matrices for the height and width dimensions.
 – Separate 1D RoPE calculations are performed for the vertical and horizontal dimensions; the resulting rotations are then combined.
 – These rotation (or “rotary”) operations are applied directly to the Q and K vectors within the masked multi-head self-attention (MHSA) layers. This ensures that positional information modulates the attention scores exactly as intended by the FiT paper.
 – For handling resolutions beyond the training distribution, scaling factors (derived per the paper’s formulas and the config’s scale factor rule: s = max(max(H_test, W_test)/L_train, 1.0)) are computed and applied at the time of computing these rotations. Rather than referring to undefined helpers (like _adjust_rotary_embeds), the integration is described as part of the internal attention sub-module where the Q and K vectors are rotated using explicit formulas from the paper.

────────────────────────────────────────────
5. TRANSFORMER BLOCKS WITH MODIFIED COMPONENTS

A. Masked Multi-Head Self-Attention (MHSA)
 – Each transformer block uses a masked MHSA module. An attention mask is constructed such that “real” tokens receive 0 while padded tokens are assigned –∞, preventing them from contributing to the softmax calculation.
 – Before computing scaled dot-product attention, the MHSA module applies the 2D RoPE rotary transformation to both the query and key vectors as described. This ensures that the positional information is inherent to the attention calculations.

B. FFN with SwiGLU
 – The standard MLP in the feed-forward network is replaced by a SwiGLU-based block following the formula:
  f(x) = Linear( SiLU(W₁ * x) ⊙ (W₂ * x) )
 – This block is integrated with standard residual connections and layer normalization.

C. Stacking Transformer Layers
 – A series of these modified blocks (combining masked MHSA with rotary modifications and the SwiGLU-based FFN) are stacked, with layer count, hidden size, and attention head configurations matching the DiT baseline.

6. DIFFUSION-SPECIFIC CONDITIONING IN THE FORWARD PASS

• The forward method now explicitly includes a diffusion-conditioning argument (e.g., “t”) to capture timestep or noise embedding details required for the denoising process.
 – The forward(x: Tensor, t: Tensor) method first processes the input image batch x by encoding it via the pretrained VAE.
 – The diffusion-conditioning input “t” is then utilized to compute appropriate conditioning embeddings, which are combined with the token representations prior to transformer processing. This combination may occur via a learned linear projection or by addition to the token embeddings.
 – These details are critical to ensure that the diffusion process (i.e., prediction of the denoised latent representation) is properly conditioned on the noise timestep.
 – After processing through the transformer, the forward method outputs the predicted denoised latent tokens that will be used subsequently for loss computation (and eventually for reconstructing images via a private decode routine).
