• The load_pretrained_vae(vae_path: str) method leverages Hugging Face or equivalent APIs to load the pretrained VAE.
• The VAE is used in two ways:
  - Its encoder converts input images (preprocessed to ensure that HW does not exceed 256×256) into latent representations.
  - Its decoder converts processed latent tokens back into images during inference.

3. Latent Tokenization and Patchification (patchify_and_pad)

• Patchification Strategy:
  - The VAE encoder produces a latent feature map with shape [B, C, H, W]. The patchification routine divides each latent map into non-overlapping patches using the patch size (p = 2). For example, if H and W are the height and width of the latent map, then the number of patches is computed as:
      H_patches = floor(H / patch_size) and W_patches = floor(W / patch_size)
      Total tokens = H_patches * W_patches.
  - Each token is obtained by flattening the patch (with dimension C × patch_size × patch_size).

• Token Padding and Truncation Details:
  - After patchification, the resulting sequence length L may vary per sample due to differences in H and W.
  - If L is less than or equal to the maximum token length (256):
      • The sequence is padded with a chosen pad token (e.g. zeros) until its length reaches exactly 256.
      • A binary mask is created in which indices corresponding to real tokens are marked as valid (e.g., True) and the added padding positions as invalid (e.g., False or zero). This mask will later be applied during the attention step.
  - If L exceeds 256:
      • The sequence is truncated; that is, only the first 256 tokens are retained. In our baseline implementation, “first 256 tokens” are determined using a row-major ordering of patches.
      • The validity mask is generated for these retained tokens only (all marked as valid), and no padding is necessary since the sequence reaches the fixed length.
  - The patchify_and_pad() function will thus robustly handle both situations. It will:
      • Accept a latent tensor from the VAE.
      • Reshape and flatten the patches into a token sequence.
      • Determine the token count: if token_count < 256, pad the remainder; if token_count > 256, truncate to 256.
      • Return the processed fixed-length token tensor along with the corresponding validity mask.

• Positional Embedding Consideration in Patchification:
  - Alongside the token sequence, the spatial coordinates for each token are computed (for example, via grid coordinates based on patch order). These coordinates are used as inputs for computing 2D RoPE.
  - The same truncation or padding strategy is applied to the positional embedding grid so that it exactly matches the final token sequence length (256 tokens).

4. Transformer Backbone Construction

• The transformer is built by stacking multiple transformer blocks. Each block is tailored to process variable token lengths (though, after patchification, the batch is fixed to 256 tokens per sample):
  
  A. 2D RoPE for Positional Embeddings:
     - Instead of fixed (absolute) positional embeddings, the implementation computes 2D rotary positional embeddings using the (x, y) positions derived from patchify_and_pad.
     - Two independent 1D RoPE embeddings are computed: one for height and one for width. They are concatenated to form a full 2D positional embedding for each token.
     - The implementation uses a rotary base (b = 10000) and a scaling factor (s), dynamically computed as s = max(max(H_test, W_test)/L_train, 1.0) when dealing with out-of-distribution resolutions. (L_train is defined as √(L_max), i.e. √256.)
     - Optionally, a decoupled variant can be implemented where separate rotary frequencies (Θ_h and Θ_w) are computed.

  B. Masked Multi-Head Self-Attention (MHSA):
     - Each transformer block includes a self-attention module that uses an attention mask M.
     - The mask M is built from the validity mask generated during patchification. Valid tokens receive 0 and padding tokens (if any) receive –∞ prior to the softmax operation.
  
  C. Feed-Forward Network with SwiGLU:
     - Replace the typical two-layer MLP with a SwiGLU block. This involves linear projections where one branch is passed through a SiLU activation and then combined element-wise with another linear transformed branch.
  
  D. Residual Connections and Layer Normalization:
     - Each block follows the pattern of applying layer normalization before the attention and FFN sublayers, with residual connections added afterward.

5. Positional Interpolation Methods for Resolution Extrapolation

• Beyond the basic computation of 2D RoPE during training, the model supports multiple interpolation strategies to adapt positional embeddings when generating images at resolutions outside the training set:
  
  - PI (Position Interpolation): Rescales position coordinates linearly to the training range.
  - EI (Embedding Interpolation): Uses bilinear interpolation to resize the positional embedding tensor.
  - NTK/YaRN and VisionNTK/VisionYaRN: Adjust the rotary frequencies directly using scaling factors. For YaRN, incorporate a ramp function γ(r(d)) with hyperparameters α = 1 and β = 32; for Vision variants, compute separate scale factors for height and width.

• Each method is implemented as a separate function (or class method) that can be invoked during inference or when the model is applied to resolutions that differ from that of training.

6. Forward Function and Diffusion Integration

• The forward(x: Tensor) method orchestrates the entire forward pass:
  
  A. Input Processing:
     - The input tensor (raw images) is passed to the VAE encoder if required.
     - The resulting latent representation is passed to patchify_and_pad(), which outputs a fixed-length sequence of tokens and a validity mask.
  
  B. Positional Embedding Addition:
     - Positional embeddings are computed using the 2D RoPE mechanism based on patch grid coordinates.
     - Should the inference resolution fall outside the training domain, the appropriate positional interpolation method (PI, EI, NTK, YaRN, VisionNTK, or VisionYaRN) is applied to adjust these embeddings.
  
  C. Transformer Block Processing:
     - The token embeddings (optionally after an input projection) are summed with the positional embeddings.
     - This combined representation is then passed through the stacked transformer blocks, where masked MHSA and SwiGLU-based FFNs are applied in sequence.
  
  D. Output:
     - The network outputs a sequence of latent tokens that are used either for computing the diffusion loss (by comparing against a target, only on valid tokens using the mask) or for reconstructing images via the VAE decoder during inference.

7. Data Flow Recap and Interaction

• Data Flow Overview:
  1. Input images → preprocessor → VAE encoder → latent representation.
  2. Latent representation → patchify_and_pad produces:
     - A fixed-length token sequence (256 tokens) using padding when the sequence is too short and truncation (row-major order) when too many tokens are present.
     - A corresponding token validity mask.
  3. Patch grid coordinates → 2D RoPE → positional embeddings (adjustable via interpolation methods if needed).
  4. Combined token and positional embeddings are processed through transformer blocks (with masked MHSA and SwiGLU FFN).
  5. Output tokens → used for diffusion loss computation or decoded into images by the VAE decoder.

• Throughout the pipeline, configuration parameters from config.yaml (e.g., patch_size, max_token_length, EMA decay, specific diffusion hyperparameters) are referenced to maintain consistency.
