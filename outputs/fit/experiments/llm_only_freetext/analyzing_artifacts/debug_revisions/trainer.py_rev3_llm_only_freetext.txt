Training Loop (train method):
2. Iterate Over Batches:
   • For each training iteration (up to the configured number of steps):
     - Retrieve a batch from the DataLoader. Each batch contains:
         ▪ Latent tokens derived from images via the pretrained VAE (patchified with patch size p = 2).
         ▪ A validity mask indicating which tokens are “real” and which are padding (tokens padded to L_max = 256).
     - Move the batch data (latent tokens, masks, etc.) to the same device as the model.
     - Set the model into training mode (using model.train()).

3. Diffusion Process and Loss Computation:
   • For each batch, perform the following steps:
     - (a) Noise Injection:
         ▪ For every sample in the batch, randomly sample a diffusion timestep t from a discrete uniform distribution over [1, T] (where T is the total number of diffusion timesteps, defined in the DDPM noise schedule).
         ▪ For each sample’s latent representation x, sample noise ε from a standard normal distribution N(0, I). 
         ▪ Use the diffusion equation to generate the noisy latent tokens x_t:  
              x_t = sqrt(ᾱ_t) * x + sqrt(1 - ᾱ_t) * ε  
           where ᾱ_t (often computed as the cumulative product of (1 − β_t) over timesteps) is a known scaling factor from the DDPM schedule. This precise computation ensures that the correct amount of noise is injected, following the DDPM protocol.
         ▪ The injected noise ε is then used both for generating x_t and also serves as the target for the model’s prediction.
     - (b) Forward Pass:
         ▪ Pass the noisy latent tokens x_t (and, if needed, the diffusion timestep t as an auxiliary input) through model.forward(). The model returns predictions for the noise (or the denoised tokens).
     - (c) Loss Computation:
         ▪ Utilize the validity mask provided with the batch to ensure that loss is computed only on non-padding tokens.
         ▪ Calculate the mean squared error (MSE) loss between the model’s predicted noise and the actual sampled noise ε.
         ▪ Sum the loss over the tokens, then average by the number of valid tokens to normalize the error.

4. Backpropagation and Optimizer Update:
   • Zero out the gradients using optimizer.zero_grad().
   • Call loss.backward() to compute gradients.
   • Perform optimizer.step() to update the model’s weights.
   • These operations are all executed on the proper device.

5. EMA Update:
   • After optimizer updates, update the EMA parameters:
       - For every model parameter and its EMA copy, execute the update:
            EMA_param = (ema_decay * EMA_param) + ((1 - ema_decay) * param)
       - This EMA update uses the decay value from config (0.9999) and is computed on the same device.

6. Logging & Checkpointing:
   • At configured intervals (for example, after a set number of iterations):
       - Log key training metrics such as the current loss, the number or ratio of valid tokens used in the loss, and other diagnostic metrics.
       - Save a checkpoint that includes:

           ▪ The EMA state, and
           ▪ The current training step count.
       - This facilitates later resumption if training is interrupted.
