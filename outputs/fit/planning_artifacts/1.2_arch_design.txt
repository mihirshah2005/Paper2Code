[Implementation approach]
We will utilize PyTorch as the deep learning framework along with torchvision for image transformations and data loading, and Hugging Face’s pretrained VAE module for encoding and decoding images. The system is modularized into separate components for dataset loading, model construction (incorporating the FiT transformer with flexible tokenization, 2D RoPE, Masked MHSA, and SwiGLU), training (including diffusion process scheduling, EMA updates, and loss computed only on valid tokens), and evaluation (calculating metrics such as FID, IS, sFID, Precision, and Recall). Open-source libraries will be used to keep the code concise and maintainable.

[Data structures and interfaces]
classDiagram
    class Main {
        +__init__(config: dict)
        +run_experiment() -> None

        +__init__(config: dict)

        +__init__(model: Model, dataloader: DataLoader, config: dict)
        +evaluate() -> dict
    }

[Program call flow]
sequenceDiagram
    participant M as Main
    participant DL as DatasetLoader
    participant MD as Model
    participant TR as Trainer
    participant EV as Evaluation

    M->>DL: Create DatasetLoader(config) and call load_data()
    DL-->>M: Return DataLoader with transforms, resizing, and token padding
    M->>MD: Instantiate Model(params) 
    MD->>MD: Load pretrained VAE via load_pretrained_vae(vae_path)
    M->>TR: Initialize Trainer(model, dataloader, config)
    TR->>MD: For each training batch, perform forward(x), compute loss (masking padding tokens), and update weights (using AdamW and EMA)
    TR-->>M: Training completes and checkpoints are saved
    M->>EV: Create Evaluation(model, dataloader, config) and call evaluate()
    EV->>MD: Process evaluation batches through forward()
    MD-->>EV: Return latent tokens/predictions
    EV-->>M: Return evaluation metrics (FID, IS, sFID, Precision, Recall)
