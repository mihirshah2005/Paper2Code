2. Method: load_data()
────────────────────────────
Objective:
• Load raw datasets according to the domain (CV, NLP, or graph), perform dataset splitting, and apply any experimental preprocessing (noise or imbalance) as dictated by the configuration and dataset identifier.

Detailed Logic:
a. Domain and Identifier Parsing:
  – Take the input dataset identifier (e.g., “cifar10-noise-0.4”) and parse it to determine:
   • The core dataset name (e.g., “cifar10”, “cifar100”, “tinyimagenet”, etc.).
   • Whether any noise (“noise-p”) or imbalance (“imbalance-r”) settings are specified.
  – Use a mapping or pre-defined list to infer the domain (CV, NLP, Graph). For instance, if the identifier contains “cifar” or “tinyimagenet,” the domain is CV; if it matches known GLUE tasks, the domain is NLP; if it corresponds to “ogbg” or TUDataset names, the domain is Graph.

b. Loading the Dataset:
  – For CV datasets:
   • Use torchvision.datasets (or a custom loader for Tiny-ImageNet) to download and load CIFAR-10, CIFAR-100, or Tiny-ImageNet.
   • Split the training set into training and validation using a 9:1 ratio. Use the provided test sets (or the official validation set for Tiny-ImageNet) as the test set.
  – For NLP datasets:
   • Load the GLUE benchmark datasets (excluding problematic sets such as WNLI) using tools like Hugging Face datasets.
   • Because test labels are not provided for GLUE tasks, use the validation set as a surrogate test set.
  – For Graph datasets:
   • For ogbg-molhiv, use the official splits available directly from the OGB package.
   • For additional graph datasets from TUDataset, perform a random split, using an 8:1:1 ratio for training, validation, and test.

c. Preprocessing the Training Set:
  – If the parsed dataset identifier indicates a noise setting (e.g., includes “noise-0.4”), extract the noise probability (p) and call apply_noise() on the training portion.
  – If an imbalance setting is specified (e.g., “imbalance-50”) and the domain is CV, extract the imbalance factor (r) and call apply_imbalance() on the training set.
  – If both settings are specified, apply noise injection first and then imbalance (or vice versa) based on documented experimental requirements.

d. Final Return Value:
  – Return a dictionary (or similar data structure) containing the “train”, “val”, and “test” splits. These splits should be in a format that can be easily wrapped in torch DataLoaders by the Training pipeline.

3. Method: apply_noise(data: Any, p: float)
────────────────────────────
Objective:
• Inject label noise into the training set by randomly flipping a percentage (p) of sample labels.

Detailed Logic:
a. Iterate through each sample in the training dataset.
  – For each sample, generate a random number (using a reproducible random generator such as numpy’s random module, with seeds controlled by the reproducibility settings).
  – If the random number is less than the specified noise probability p, replace the current label with a different one.
  – The new label is chosen uniformly at random from the set of available class labels (excluding the current label to avoid no-op changes).
b. Return the training dataset with the modified labels.

────────────────────────────
4. Method: apply_imbalance(data: Any, r: int)
────────────────────────────
Objective:
• Resample the training data to create a long-tailed distribution, where class sample counts follow an exponential decay.

Detailed Logic:
a. Validate the Context:
  – Confirm that the dataset belongs to the CV domain (as the imbalance setting is primarily applied here). If not, simply return the dataset without modification.
  – If the imbalance factor r is 1 (indicating no imbalance), return the original dataset.
b. Compute Target Sample Counts:
  – Determine the number of classes and assume the original dataset is balanced.
  – Calculate a decay factor d based on the relationship: r = n₀ / n₍C-1₎, with d computed as (1/r)^(1/(C-1)).
  – For each class c (from 0 to C-1), calculate the target number of samples as n_c = n₀ * d^c (using rounding or flooring as appropriate).
c. Resample Data:
  – For each class, randomly select n_c samples from that class.
  – Use a reproducible random sampling method to ensure that repeated experiments yield the same imbalanced set.
d. Return the re-sampled (imbalanced) training dataset.
