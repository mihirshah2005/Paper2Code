 – Responsibilities:  
  – Store the experiment configuration (loaded from config.yaml) for later use.  
  – Select the appropriate model architecture based on the domain (CV, NLP, or Graph) and the provided model_name.  
  – Adjust the model’s input/output layers according to dataset properties (for example, the number of classes or input dimensions).

 – Inputs:

  • dataset_info: A dictionary supplying dataset-specific properties such as the number of classes, input feature dimensions, or image channels.  
 – Operation Overview:  
  • The method first inspects the domain parameter.

     • Construct a Graph Convolutional Network using torch_geometric’s GCNConv layers, followed by a global pooling and a final linear layer.

     • Build a Graph Attention Network using GATConv layers, ensuring the final output dimension equals dataset_info["num_classes"].

     • Instantiate a Graph Isomorphism Network using GINConv layers and appropriate aggregation, then adjust the final classification layer to output dataset_info["num_classes"].

  • The method returns the fully constructed and adjusted nn.Module instance that is ready for integration into the training loop.

• Domain Detection and Routing:  
 – The function first identifies which branch to take using the value of the domain parameter.  
 – For each domain, supported model names are checked; if the provided model_name is not in the supported list, an error is raised immediately to prevent further execution.  
 – This explicit error handling ensures that any misconfiguration or typo in the parameters leads to immediate feedback rather than failing silently later.

• Dataset Adjustments:  
 – The final classification layer is set to match dataset_info["num_classes"].  
 – CV models might also need adjustments in the first layer if image channels differ from defaults (typically 3 channels).

• Pretrained Weights Usage:  
 – For NLP models (BERT, GPT2), pretrained weights are loaded via Hugging Face Transformers.  
 – The other models are instantiated from scratch, using either torchvision utilities or custom implementations.
