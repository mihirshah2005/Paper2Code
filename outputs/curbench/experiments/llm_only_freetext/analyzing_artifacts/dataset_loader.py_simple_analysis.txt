────────────────────────────
Objective:

Detailed Logic:
a. Domain and Identifier Parsing:
  – Parse the input dataset identifier (for example, “cifar10-noise-0.4”) to determine the core dataset name and whether noise ("noise-p") or imbalance ("imbalance-r") processing is needed.
  – Identify the domain by matching the core name against known CV datasets (e.g., “cifar10”, “cifar100”, “tinyimagenet”), NLP tasks (e.g., GLUE tasks), or graph datasets (e.g., “ogbg-molhiv” or other TUDataset names).
  – If the identifier is not recognized or is incomplete, the loader should raise a clear error message or, if possible, fallback to a default (for example, defaulting to the standard setting without noise/imbalance).
  
b. Loading the Dataset:
  – For Computer Vision (CV):
   • Use torchvision.datasets (or a custom loader for Tiny-ImageNet) to download and load datasets such as CIFAR-10, CIFAR-100, or Tiny-ImageNet.
   • Split the training set into a new training set and a validation set with a 9:1 ratio, and choose the appropriate test set based on the dataset’s specification.
  – For Natural Language Processing (NLP):
   • Use Hugging Face’s datasets setup to load GLUE benchmark datasets (excluding problematic ones) and assign the validation set as the test set since test labels are not provided.
  – For Graph datasets:
   • For ogbg-molhiv, rely on the official split provided by OGB.

c. Preprocessing the Training Set:
  – If the dataset identifier contains a noise parameter (indicated by “noise-p”), extract the noise probability (p) and call the apply_noise() method on the training split.
  – If the identifier contains an imbalance parameter (indicated by “imbalance-r”) and the data domain is CV, extract the imbalance factor (r) and call apply_imbalance() on the training set.
  – If multiple modifications are specified, define a clear order (for example, applying noise first before rebalancing) and document this ordering.
  
d. Return Value:
  – Package the processed datasets into a dictionary with keys such as "train", "val", and "test", ready to be converted into PyTorch DataLoaders within the training module.

3. Method: apply_noise(data: Any, p: float)
────────────────────────────
Objective:
• To add label noise to the training set by randomly altering the labels of a specified percentage of samples.

Detailed Logic:
a. Iteration and Random Decision:
  – Loop through each sample in the training dataset.
  – For each sample, generate a random number (using reproducible methods like numpy’s random generator set with the correct seed from config.reproducibility.seeds).
  – If the random value is less than p, change the sample’s label.
  
b. Changing the Label:
  – The new label is chosen uniformly at random from the available class labels, ensuring that it differs from the sample’s original label.
  
c. Return the Noisy Dataset:
  – After processing all samples, return the modified dataset to be used during training.

4. Method: apply_imbalance(data: Any, r: int)
────────────────────────────
Objective:
• To adjust the training data distribution to mimic a long-tailed (imbalanced) dataset by resampling class data according to an exponential decay function.

Detailed Logic:
a. Validate the Context:
  – Confirm that this operation is only applied when the data domain is CV. For NLP or graph datasets, simply return the dataset unchanged.
  – If the imbalance factor r is 1 (i.e., the dataset is balanced), return the original dataset.
  
b. Compute Target Counts:
  – Determine the number of classes (C) in the dataset and assume the original training set is balanced.
  – Compute the decay factor d from the relationship r = n₀/n₍C-1₎, typically taking d = (1/r)^(1/(C-1)).
  – For each class c from 0 to C-1, calculate the target sample count n_c = n₀ * d^c (using appropriate rounding).
  
c. Resample the Dataset:
  – For each class, randomly pick n_c samples from that class using a reproducible random sampling method.
  – Merge the resampled data to create the imbalanced training set.
  
d. Return the Imbalanced Dataset:
  – Provide the newly formed training set, now following a long-tailed distribution, for downstream use.
