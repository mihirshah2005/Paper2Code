  – Detect any modifiers attached to the identifier, such as “noise-0.4” and “imbalance-50”.

 b. Loading the raw dataset using domain‑appropriate libraries:

   • Use torchvision.datasets for CIFAR‑10 and CIFAR‑100.

  – For non‑CV domains (NLP and Graph):

 d. Upon completion, load_data() returns a dictionary with keys “train”, “val”, and “test”. This return format—whether as raw Dataset objects or wrapped in DataLoaders—is uniform across domains for seamless integration with downstream modules (e.g., training and evaluation).

3. apply_noise(data: Any, p: float) Method

• Purpose: To inject uniform random noise into the training labels.  
• Logic:  
  1. Iterate over each sample in the provided training dataset.  
  2. For each sample, generate a random value (using a reproducible random method such as torch.rand or numpy.random.rand).  
  3. If the random value is below the noise probability p, randomly select a new label from the class index set—ensuring that it differs from the original label.  
  4. Return the modified dataset (with noise injected only into training labels).  
• Note: This method is designed to work across domains where label noise injection is valid (e.g., CV and NLP, and under controlled scenarios for Graph data if applicable).

4. apply_imbalance(data: Any, r: int) Method

• Purpose: To create a long‑tailed version of the training dataset for CV tasks.  
• Logic:  
  1. Confirm that the dataset is for a CV domain; if not, log a warning and return the original data.  
  2. Group the training samples by their class labels.  
  3. Verify that the dataset is roughly balanced. If the class counts vary significantly from each other, log a warning: since the intended experimental design assumes balanced starting conditions, one option is to use a representative baseline count (e.g., the average or median count) instead of the maximum class count (n₀). This adjustment is documented as a deviation and should be clearly noted.  
  4. Assuming a balanced starting point, compute the decay factor d using the formula:  
    d = (1 / r)^(1 / (C – 1))  
    where C is the total number of classes.  
  5. For each class c (in ascending order), calculate the desired sample count as:  
    desired_count[c] = int(n₀ × d^c)  
  6. Randomly select the desired number of samples from each class (using a fixed random seed) and combine these selections into a new imbalanced training dataset.  
  7. Return this imbalanced dataset.  
• Note: This method is primarily intended for balanced CV datasets (e.g., CIFAR). If the input data is not balanced, the computed imbalance ratio might not strictly match the intended design. In such cases, logging and possibly an alternative baseline (using average/median counts) should be considered.
