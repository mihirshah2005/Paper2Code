[Logic Analysis]
- ['config.py', 'Config class to load and manage experiment configuration parameters (hyperparameters, dataset paths, training settings, curriculum method parameters). This file is imported by all other modules to ensure consistency.']
- ['dataset_loader.py', 'DatasetLoader class handles loading and preprocessing of data for CV, NLP, and graph domains. Includes methods load_data() to download and split datasets, apply_noise() to inject label noise, and apply_imbalance() to create long-tailed distributions. Depends on torch, torchvision, transformers (for NLP), and torch-geometric.']
- ['model.py', 'ModelManager class defines and builds backbone models based on domain. Contains methods to instantiate models (LeNet, ResNet-18, ViT for CV; LSTM, BERT, GPT2 for NLP; GCN, GAT, GIN for Graph). Uses PyTorch modules and Hugging Face Transformers. Adjusts input and output layers based on dataset properties.']
- ['curriculum.py', 'CurriculumScheduler abstract class defines the interface for curriculum learning strategies. Concrete subclasses implement specific methods (e.g., Self-Paced Learning (SPL), TTCL, loss reweighting methods). Provides methods such as update_schedule(loss: float, epoch: int) to compute data weights or selection criteria. Imports necessary math and PyTorch functions.']
- ['trainer.py', 'Trainer class encapsulates the training loop. Initializes the model, optimizer, loss function, and integrates the CurriculumScheduler to adjust training dynamics based on current loss and epoch. Responsible for running training epochs, logging training progress, and tracking GPU time/memory usage. Dependent on dataset_loader, model, curriculum, and config.']
- ['evaluation.py', 'Evaluation class defines the evaluation logic. Implements methods to perform forward passes through the trained model to compute performance metrics (accuracy, F1 score, correlations, AUC) based on domain-specific criteria. Uses functions from scikit-learn and PyTorch. Outputs a summary dictionary of metrics.']
- ['main.py', 'Main entry point that ties together configuration, data loading, model building, curriculum scheduler instantiation, training, and evaluation. Orchestrates experiment runs with multiple seeds for reproducibility and logs final outputs. Depends on all previously mentioned modules.']
