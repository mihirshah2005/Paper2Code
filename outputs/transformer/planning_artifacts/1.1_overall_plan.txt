Below is a detailed roadmap that covers the key elements from the Transformer paper, outlining the methodology, dataset and experimental requirements, hyperparameters, training settings, and evaluation procedures. This plan is designed to guide a step‐by‐step reproduction of the experiments described in “Attention Is All You Need.”

─────────────────────────────  
1. Overview of the Methodology

 • Architecture:  
  – The model consists of separate encoder and decoder stacks.  
  – Both encoder and decoder are built from N identical layers (N = 6 for the base model).  
  – Each encoder layer has two sub-layers:  
   ▪ A multi-head self-attention mechanism  
   ▪ A position-wise feed-forward network  
  – Each decoder layer has three sub-layers:  
   ▪ Masked multi-head self-attention (to prevent attending to future positions)  
   ▪ Multi-head attention over the encoder’s output (encoder-decoder attention)  
   ▪ A position-wise feed-forward network  
  – Residual connections and layer normalization are applied around each sub-layer.

 • Attention Mechanism:  
  – The paper introduces Scaled Dot-Product Attention, where for a set of queries Q, keys K, and values V the attention weights are computed via:  
   Attention(Q, K, V) = softmax(QKᵀ/√dₖ) V  
  – Multi-head attention is employed by linearly projecting Q, K, and V h = 8 times (with dₖ = dᵥ = d_model/h = 64) and then concatenating the results before applying a final linear projection.

 • Feed-Forward Network:  
  – Each layer contains a two-layer feed-forward network with a ReLU in between.  
  – For the base model, the input/output dimensionality is d_model = 512 and the inner hidden layer has d_ff = 2048.

 • Positional Encoding:  
  – Since the model has no recurrence or convolution, positional information is injected using sinusoidal positional encodings computed as:  
   PE(pos, 2i) = sin(pos / 10000^(2i/d_model))  
   PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))  
  – An alternative using learned positional embeddings was also found to yield similar results.

 • Embedding Sharing:  
  – The same weight matrix is used both for input token embeddings and the pre-softmax linear transformation.  
  – The embeddings are scaled by √d_model.

2. Dataset and Experimental Requirements

 A. Translation Tasks  
  • Datasets:  
   – WMT 2014 English–German (≈4.5M sentence pairs)  
   – WMT 2014 English–French (≈36M sentence pairs)  
  • Preprocessing:  
   – Use subword tokenization:  
    ▪ For En–De, apply byte-pair encoding with a shared vocabulary of approximately 37,000 tokens.  
    ▪ For En–Fr, use a word-piece vocabulary containing about 32,000 tokens.  
  • Batching:  
   – Group sentence pairs by approximate sequence length so that each batch contains roughly 25,000 tokens on both source and target sides.

 B. Constituency Parsing  
  • Datasets:  
   – Use the Wall Street Journal (WSJ) portion of the Penn Treebank (~40K training sentences) for a “WSJ only” experiment.  
   – For semi-supervised experiments, supplement with additional corpora (e.g., high-confidence and BerkeleyParser corpora totaling approximately 17M sentences).  
  • Vocabulary:  
   – For WSJ-only experiments use a vocabulary of 16K tokens; for the semi-supervised setting, use 32K tokens.

3. Hyperparameters and Training Settings

 • Model Configurations  
  – Base Model:  
   ▪ N = 6 layers for encoder and decoder  
   ▪ d_model = 512, d_ff = 2048  
   ▪ h = 8 attention heads (with dₖ and dᵥ = 64 each)  
   ▪ Dropout rate (P_drop) = 0.1 applied after each sub-layer and on the summed embeddings + positional encodings  
   ▪ Label smoothing: ϵ_ls = 0.1 during training  
  – Big Model (for translation):  
   ▪ Increased dimensions (e.g., d_model = 1024, d_ff = 4096)  
   ▪ Dropout adjustments: e.g., P_drop = 0.1 for English–German and possibly 0.1 (instead of 0.3) for English–French tasks.

 • Learning Rate Schedule  
  – Use the Adam optimizer with:  
   ▪ β₁ = 0.9, β₂ = 0.98, ε = 1e-9  
  – Learning rate schedule:  
   lrate = d_model^(–0.5) * min(step_num^(–0.5), step_num*(warmup_steps^(–1.5)))  
  – Warmup steps: 4000

 • Training Timings and Checkpointing  
  – Base model:  
   ▪ Train for 100,000 steps (approximately 12 hours on 8 NVIDIA P100 GPUs with a step time of ~0.4 sec)  
   ▪ Average the last 5 checkpoints (saved at ~10-minute intervals) for inference  
  – Big model:  
   ▪ Train for 300,000 steps (approximately 3.5 days on 8 GPUs, ~1.0 sec per step)  
   ▪ Average the last 20 checkpoints  
  – Batch size: Approximately 25,000 source tokens and 25,000 target tokens per batch

 • Inference Settings for Translation  
  – Use beam search:  
   ▪ Beam size = 4  
   ▪ Length penalty α = 0.6  
   ▪ Maximum output length: input length + 50 (with early termination if possible)

 • Inference Settings for Parsing  
  – Use beam search with:  
   ▪ Beam size = 21  
   ▪ Length penalty α = 0.3  
   ▪ Maximum output length: input length + 300

   – Code the sinusoidal positional encoding function as defined.

  3. Multi-Head Attention Layer  
   – For each attention head, implement:  
    ▪ Linear projections for Q, K, and V (with parameters W_Q_i, W_K_i, W_V_i)  
    ▪ Scaled dot-product attention computation  
   – Concatenate the outputs from all heads and apply a final linear projection (W_O).  
   – Implement masking in the decoder’s self-attention to prevent attending to future positions.

  4. Position-wise Feed-Forward Network  
   – Implement two linear layers with a ReLU activation between.  
   – Use the dimensions d_model for input/output and d_ff for the hidden layer.

  5. Encoder and Decoder Layer Blocks  
   – Create an encoder layer that chains together the multi-head self-attention and feed-forward network with residual connections and layer normalization.  
   – Similarly, build the decoder layer with its three sub-layers (masked self-attention, encoder-decoder attention, and feed-forward network), applying residual connections and layer normalization for each.

  6. Full Transformer Model  
   – Assemble the encoder stack and decoder stack.  
   – Add the shared input embedding layers and the final linear+softmax layer (with tied weights to the embeddings).

  7. Training Loop  
   – Set up the training loop using a deep learning framework (e.g., PyTorch or TensorFlow).  
   – Integrate the Adam optimizer with the custom learning rate scheduler as specified.

  8. Inference and Beam Search

  9. Logging and Checkpointing
