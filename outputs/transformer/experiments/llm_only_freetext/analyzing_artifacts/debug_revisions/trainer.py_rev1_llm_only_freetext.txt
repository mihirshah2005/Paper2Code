• The Trainer must update the optimizer’s learning rate at each training step using the formula from the paper:
  learning_rate = d_model^(–0.5) * min(step_num^(–0.5), step_num * warmup_steps^(–1.5))

• At each training step:
  – Calculate the new learning rate based on the current global step and the d_model and warmup_steps values read from the config.
  – Update the learning rate in the optimizer’s parameter groups (e.g., optimizer.param_groups[0]['lr'] = computed_lr).
  – This ensures that as training progresses, the learning rate first increases linearly (during the warmup phase) and then decays proportionally to the inverse square root of the step number.

  b. Forward Pass:

  c. Loss Computation:
   – Compare the model predictions to the ground truth target sequences using the loss function.

  e. Learning Rate Update:
   – Immediately after each optimizer.step(), compute the new learning rate using the custom scheduler.
   – Update optimizer.param_groups accordingly.

  f. Logging and Metrics:

   – Save the current model state, optimizer state, training step counter, and any additional metadata.
