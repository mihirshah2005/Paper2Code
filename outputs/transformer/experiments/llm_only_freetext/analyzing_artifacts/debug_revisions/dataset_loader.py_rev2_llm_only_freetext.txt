6. Dynamic Batching Calculation and Collation Strategy  
 • A key requirement is that each batch approximately contains 25,000 tokens on the source side and 25,000 on the target side (as per the config).  
 • Two common strategies can be employed:  
  1. Sum-Based Accumulation:  
   – Accumulate examples until the total sum of token counts (the actual lengths of the tokenized sentences) reaches the token threshold.  
   – This method uses the actual token counts, which may vary because of different sentence lengths.  
  2. Maximum-Length Estimation:  
   – In a padded batch, the effective token count is the maximum sentence length in the batch multiplied by the number of examples.  
   – Compute the maximum length (L_max) among the examples in a candidate batch and ensure that L_max × (number of examples) does not exceed the threshold.  
 • The analysis now clarifies that the batch collation function (collate_fn) should select and group examples based on one of these methods; for instance, when processing a bucket sorted by lengths, choose a group such that  
   number_of_examples ≤ floor(25000 / L_max)  
  or target a cumulative sum of actual token counts that does not exceed 25000.  
 • This dynamic batching approach optimizes GPU memory utilization while ensuring efficiency, and it should be clearly documented in the collate_fn’s logic.
