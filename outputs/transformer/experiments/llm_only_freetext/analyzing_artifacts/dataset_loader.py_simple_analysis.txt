6. Dynamic Batching Calculation and Collation Strategy  
 • To meet the requirement that each batch roughly contains 25,000 tokens on both the source and target sides, the implementation uses a maximum-length estimation strategy.  
 • Here’s how the strategy works:  
  – When forming a batch, the collate function first gathers a candidate group of examples (from a pre-sorted bucket by length, if applicable).  
  – For the source side, compute L_source_max as the maximum token length of the source sentences in that group. Similarly, compute L_target_max as the maximum token length of the target sentences.  
  – Since deep learning frameworks generally pad sequences to the maximum length in the batch, the effective token count for the source becomes:  
   Effective_source_tokens = L_source_max × (number of examples)  
  and similarly for the target:  
   Effective_target_tokens = L_target_max × (number of examples)  
  – The collate function then enforces that both effective counts are less than or equal to the threshold (25000 tokens). This means the group is accepted as a batch only if:  
   L_source_max × (number of examples) ≤ 25000  
   and  
   L_target_max × (number of examples) ≤ 25000  
 • This maximum-length estimation method is applied independently to both source and target sides, ensuring that neither padded side exceeds the token budget.  
 • (Note: Although another sum-based accumulation method exists — which would add the actual token counts across examples — this implementation explicitly follows the maximum-length estimation strategy.
