2. Training Loop (train method)

 • Loop over training batches until reaching the total number of steps (100K for base or 300K for big). For each training iteration:
  – Set the model to training mode.
  – Retrieve a batch of source and target token sequences from the train_loader.
  – Forward Pass:  
   ▪ Feed the batch (and the corresponding attention mask—for instance, for decoder self-attention masking) to the model’s forward() method.
   ▪ Obtain the predicted output logits.
  – Compute the Loss:  
   ▪ Use a cross-entropy loss function augmented with label smoothing (with ε_ls = 0.1 taken from config.training.label_smoothing).  
   ▪ The loss function should appropriately handle the padded tokens.
  – Backward Pass and Optimization:  
   ▪ Perform back-propagation (loss.backward()).  
   ▪ Call optimizer.step() to update the model parameters.
   ▪ Immediately after the optimizer step, update the learning rate scheduler (i.e., scheduler.step()).  
   ▪ Zero out gradients for the next iteration using optimizer.zero_grad().
  – Increment the global training step counter.
  – Learning Rate Scheduling:  
   ▪ At each step, the lambda function computes the new learning rate according to:  
    lr = d_model^(-0.5) * min(step^(-0.5), step * warmup_steps^(-1.5))  
   ▪ Even if step=0 is not a valid input, ensure that you set a minimal counter (e.g., treat the first step as 1) to avoid division by zero.
  – Logging Metrics:  
   ▪ Measure and log the current training loss, and compute perplexity, for example as exp(loss).  
   ▪ Optionally log the current learning rate.
  – Periodically Evaluate on Validation Set:  
   ▪ Every fixed interval (e.g., every N steps or based on elapsed time) switch the model to evaluation mode, run the validation set through the model (possibly using a separate Evaluation class for inference) to compute metrics such as BLEU score (for translation) or perplexity, and log these metrics.
  – Checkpoint Saving:  
   ▪ Monitor the elapsed training time (using Python’s time module) and, if the elapsed time since the last checkpoint save exceeds config.checkpoint.checkpoint_interval_minutes (10 minutes), call the save_checkpoint() method.  
   ▪ Maintain a list or queue of recent checkpoint file paths to later perform checkpoint averaging.
   ▪ Use the config parameter average_checkpoints (5 for base, 20 for big) during final model averaging before inference.
