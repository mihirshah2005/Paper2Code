• Input Embeddings:  
 – The model converts source (and target) token indices to vector embeddings.  
 – The embeddings are scaled by √(d_model).  
 – Positional encodings (sinusoidal by default) are computed and added elementwise to the token embeddings.  

• Weight Tying Considerations:  
 – By default, a single embedding layer is shared between the encoder and decoder, and its weights are tied to the final linear projection (output layer).  
 – However, if the task involves different vocabularies (as can be the case in some translation tasks), the design should allow for separate embedding matrices (for source vs. target) and deactivate weight tying. This behavior can be controlled via a configuration flag.

3. ENCODER AND DECODER MODULES

A. Encoder  
 • The Encoder sub-module instantiates “num_layers” encoder layers.  
 • Each encoder layer comprises:  
  1. Multi-head self-attention using the MultiHeadAttention helper class.  
   – It performs linear projections (for Q, K, V) for each head.  
   – Computes scaled dot-product attention as:  
    Attention(Q, K, V) = softmax((QKᵀ / √d_k) + mask) * V  
   – Here, the mask includes any padding masks for the source sequence.  
   – The mask is expected to have shape [batch_size, 1, 1, src_seq_len] (or a compatible shape) so that it can be broadcasted across all attention heads and query positions.  
  2. Residual connection and layer normalization follow the attention computation.  
  3. A position-wise feed-forward network (via PositionwiseFeedForward) follows with two linear layers (d_model → d_ff and d_ff → d_model), with ReLU and dropout.  
  4. A final residual connection and layer normalization are applied.

B. Decoder  
 • The Decoder sub-module similarly instantiates “num_layers” decoder layers.  
 • Each decoder layer contains three sub-layers:  
  1. Masked multi-head self-attention:  
   – In addition to a padding mask (to ignore padded tokens), this layer applies a subsequent (or future) mask to prevent a position from attending to future tokens.  
   – The subsequent mask is typically a lower-triangular matrix with shape [target_seq_len, target_seq_len] that is combined (e.g., via elementwise logical AND) with the target padding mask.  
   – The combined mask is then broadcasted to the shape [batch_size, num_heads, target_seq_len, target_seq_len].  
  2. Encoder-decoder attention:  
   – Here, queries come from the decoder’s output of the previous sub-layer, while keys and values are the encoder outputs.  
   – A source padding mask is applied to ignore padding positions in the encoder outputs.  
  3. Position-wise feed-forward network, similar to the encoder, applied with its own residual and normalization steps.

4. HANDLING MASKS IN THE FORWARD PASS

• The forward() method must clearly combine and apply masks for both encoder and decoder:  
 – Padding Masks:  
  ○ For the encoder, a padding mask is computed from the source sequence (e.g., where token indices are zero or a designated pad value) to prevent attention to padding tokens.  
  ○ For the decoder, a similar padding mask is computed for target tokens.  
 – Subsequent/Future Masks:  
  ○ For masked self-attention in the decoder, a subsequent mask (triangular matrix) is generated with shape [target_seq_len, target_seq_len].  
  ○ This mask is then combined with the target padding mask (using a logical AND or by addition of large negative values, ensuring masked positions in the attention logits).  
• Broadcasting Rules:  
 – Each mask, after combining, should be unsqueezed so that its dimensions conform to the attention tensor, typically [batch_size, num_heads, query_seq_len, key_seq_len].  
 – During multi-head attention, ensure that the same mask is applied for each head; this may require explicitly duplicating or broadcasting the mask over the num_heads dimension.
 – When adding the mask to the attention logits, a typical implementation adds a large negative number (e.g., -1e9) to the masked scores before applying softmax.

B. PositionwiseFeedForward  
 – Consists of two linear layers with a ReLU activation and dropout in between.  
 – Applied independently to each position in the sequence.

C. PositionalEncoding  
 – Generates a fixed sinusoidal encoding matrix based on the formula provided.
