3. Data Flow and Masking Details in DecoderLayer.forward  
• When the forward method of DecoderLayer is invoked for the decoder’s self-attention, it receives target token representations and—in addition optionally—a mask indicating positions that should be masked out (for example, to ignore padding tokens).  
• The decoder constructs an L×L causal mask for a target sequence of length L. This mask is typically built using an operation such as:  
  mask_causal = torch.triu(torch.ones(L, L), diagonal=1)  
  mask_causal = mask_causal.float().masked_fill(mask_causal == 1, float('-inf')).masked_fill(mask_causal == 0, 0.0)  
  The causal mask ensures that for every token at position i, all tokens at positions j > i are assigned a value of –∞ so that after adding the mask to the attention logits, their softmax probabilities become zero.  
• If an external mask (for example, a padding mask) is provided, it should be structured to indicate positions that must be ignored (e.g., with 0 or a boolean flag) and then converted to the same numeric format (with –∞ for masked positions and 0 for valid positions).  
• The crucial merging strategy is as follows:  
  – Convert the external mask to the same shape as the causal mask (typically [batch_size, 1, L, L]) if it isn’t already.  
  – Merge the external mask with the causal mask using element-wise addition:  
   merged_mask = causal_mask + external_mask  
  Because each mask uses –∞ for disallowed positions and 0 for allowed positions, addition guarantees that any position masked by either the causal or external mask will result in –∞ in the merged mask.  
  – The merged_mask is then added to the attention logits before applying softmax. This way, if either the causality constraint or an external condition (such as padding) requires a token to be masked, that token's attention score is set to –∞.  
• If no external mask is provided, the decoder simply uses the causal mask.  
• This merging step is critical to prevent unintended masking behavior and to ensure that the resulting mask correctly enforces both the autoregressive constraint and any additional masking (like padding token masking).
