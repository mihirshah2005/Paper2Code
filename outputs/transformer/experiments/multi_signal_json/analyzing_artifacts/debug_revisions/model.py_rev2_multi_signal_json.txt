1. Overall Structure

• The module (model.py) will house the TransformerModel class and its helper components. At the top level, TransformerModel composes:
 – An Encoder stack built from multiple EncoderLayer modules.
 – A Decoder stack built from multiple DecoderLayer modules.
 – Shared embedding layers for input (source and target tokens), with scaling by √d_model and weight tying between target embeddings and the final output projection.
 – A PositionalEncoding module which produces sinusoidal positional encodings.
 – A final linear projection layer for generating logits over the target vocabulary.
• All hyperparameters (d_model, d_ff, num_layers, etc.) are provided through configuration (config.yaml).

─────────────────────────────  
2. Components and Their Responsibilities

A. PositionalEncoding  
 – Implements sinusoidal calculations:  
  PE(pos, 2i) = sin(pos / 10000^(2i/d_model))  
  PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))  
 – Either precomputes a fixed-size coding matrix (up to a maximum sequence length) or computes it on the fly, then adds it to the token embeddings (after scaling by √d_model).

B. MultiHeadAttention  
 – Performs scaled dot-product attention across multiple heads:  
  • Projects the input into queries, keys, and values via learned linear layers.
  • Reshapes tensors to split the representation into h heads (with dimensions d_k and d_v).
  • Computes attention scores as softmax((QKᵀ) / √d_k), optionally applying a mask (see below).
  • Applies dropout to the attention weights.
  • Concatenates the outputs of each head and applies a final linear projection.
 – This module is used in both encoder self-attention and decoder layers (both masked self-attention and encoder-decoder attention).

C. PositionwiseFeedForward  
 – Implements a two-layer feed-forward network with ReLU activation:
  FFN(x) = max(0, xW₁ + b₁)W₂ + b₂.
 – Uses d_model for input/output and d_ff for the hidden layer.
 – Includes dropout applied after the activation for regularization.

D. EncoderLayer  
 – Contains two sub-layers:
  1. Multi-head self-attention.
  2. Positionwise feed-forward network.
 – Employs residual connections and layer normalization around each sub-layer:
  output = LayerNorm(input + dropout(SubLayer(input))).

E. DecoderLayer  
 – Comprises three sub-layers:
  1. Masked multi-head self-attention.
  2. Encoder–decoder attention.
  3. Positionwise feed-forward network.
 – Each sub-layer is wrapped with a residual connection and then layer normalized.
 – For the masked self-attention sub-layer, a critical step is enforcing causality:
  • An upper‐triangular mask is created such that for each target position i, positions greater than i are masked out.
  • To construct the mask, given a target sequence of length L, create an L×L matrix using a function such as torch.triu. For example, one can generate:
   mask = torch.triu(torch.ones(L, L), diagonal=1)  
   mask = mask.masked_fill(mask == 1, float('-inf')).masked_fill(mask == 0, 0.0)
  • This mask is then expanded (via unsqueeze operations) to match the dimensions of the attention scores (batch size, number of heads, L, L) so that when added to the attention logits inside MultiHeadAttention, positions that should not be attended to get a very large negative number, causing their softmax probability to be zero.
  • The mask is applied before the softmax function within the MultiHeadAttention in DecoderLayer.forward. This ensures that each position in the output only attends to positions up to and including itself, thereby preserving autoregressive behavior.

F. Encoder  
 – Applies a token embedding layer to source tokens (scaled by √d_model).
 – Adds positional encodings.
 – Processes the embeddings through a stack of EncoderLayer modules.
 – Outputs continuous representations for each input token.

G. Decoder  
 – Similar to the encoder, it first embeds target tokens and adds positional encodings.
 – Processes the target embeddings through a stack of DecoderLayer modules:
  • The first sub-layer (masked self-attention) uses the causal mask described above.
  • The second sub-layer combines the decoder activations with encoder output via multi-head attention.
 – The output is then fed to a final linear layer (with tied weights to the target embedding) to generate logits over the target vocabulary.
 – The forward method explicitly constructs and passes the causal mask when processing the masked self-attention sub-layer.

H. TransformerModel  
 – Integrates the Encoder and Decoder.
 – In its forward method, receives both source and target token sequences, along with any additional masks (e.g., padding masks for source/target).
 – Uses the encoder to compute source representations.
 – Uses the decoder to generate target representations while applying the causal mask in the first decoder layer.
 – Applies the final tied linear projection to produce the logits for each target token.
 – Can optionally return attention weights for diagnostic or visualization purposes.

3. Data Flow and Masking Details in DecoderLayer.forward

• When the forward method of DecoderLayer is called, the following occurs:
 – First, the method receives the target token representations along with an optional mask parameter.
 – Inside the masked self-attention sub-layer, if no mask is provided externally, the layer dynamically constructs an upper-triangular mask based on the sequence length:
  ○ For a target sequence with length L, generate a mask matrix using torch.triu (or equivalent) such that:
   mask = torch.triu(torch.ones(L, L), diagonal=1)
   mask = mask.float().masked_fill(mask == 1, float('-inf')).masked_fill(mask == 0, 0.0)
 – This mask is broadcast to cover the batch and head dimensions (with dimensions [batch_size, num_heads, L, L]) before being added to the attention scores.
 – The masked attention scores (after adding the mask) pass through a softmax to yield valid probability distributions that prevent "future" tokens from affecting the current token’s representation.
 – The output is then used with standard residual and layer normalization operations.
 – By incorporating the causal mask in this way, the decoder layer strictly enforces autoregressive behavior.
