  – dataset name (e.g., "Pubmed")
  – splitting method (e.g., "public")
  – For synthetic graph parameters: the compression_ratio (which is set to 0.15 with the comment “0.15% compression ratio” and the r_k value (0.9). 

• Based on the dataset name (here “Pubmed”), the loader will access the appropriate data files or online source. Although the exact file format is not specified, you can assume that the loader will:
  – Load the adjacency information (A) (perhaps as an edge list)
  – Load node features (X) (a numerical matrix)

Step 2. Compute the Normalized Laplacian

• Once the raw data is loaded, compute the graph’s degree information. For every node i, compute its degree (the sum of the corresponding row/column in A).

• Define a diagonal degree matrix D. Then, compute D^(–1/2) by inverting the square roots of each degree (taking care to handle degree-zero nodes if needed).

• Compute the normalized Laplacian L using the formula:
     L = I – D^(–1/2) * A * D^(–1/2)
  Here, I is the identity matrix of the same size as A. This computation will use NumPy (or SciPy’s sparse operations if A is large) to ensure efficiency. The resulting L must be stored (e.g., either as a dense NumPy array or a sparse matrix, depending on the dataset size).

──────────────────────────────
Step 3. Determine the Number of Synthetic Nodes and Eigenvector Count

• The synthetic graph is supposed to have a number of nodes (N′) determined by the compression ratio. With a compression_ratio from the config set to 0.15 and the comment “0.15% compression ratio”, compute:
     N = total number of nodes in the real dataset.
     N′ = ceil( (compression_ratio/100) * N )
         → For Pubmed (≈19717 nodes), N′ will be roughly 30 nodes.
         
• Set K = N′. This value K will determine how many eigenvectors to extract from the real graph.

• Using r_k (here 0.9), define:
     K1 = ceil(r_k * N′)   → the number of smallest eigenvectors (capturing low-frequency, global structure).
     K2 = N′ – K1          → the number of largest eigenvectors (capturing high-frequency, local details).
     
  This matches the paper’s requirement to select the K1 smallest and K2 largest eigenvectors, so that K1 + K2 = K.

──────────────────────────────
Step 4. Perform the Truncated Eigen-Decomposition

• For efficiency, use SciPy’s sparse eigenvalue solver (scipy.sparse.linalg.eigsh) rather than computing the full eigen-decomposition of L.

• First, compute the K1 smallest eigenvalues and corresponding eigenvectors with:
     eigsh(L, k=K1, which='SM')
  – “SM” (“smallest magnitude”) is suitable here because the normalized Laplacian eigenvalues lie in [0, 2], and we need those near 0.

• Second, if K2 > 0 (which it will be for a positive synthetic node count), compute the K2 largest eigenvalues and eigenvectors with:
     eigsh(L, k=K2, which='LA')
  – “LA” stands for “largest algebraic,” ensuring that we capture the high-frequency eigenpairs.

• Once both calls are complete, combine the eigenvalues and eigenvectors. The combined result should follow the order defined in the paper:
     U_K = [u₁, …, u_K₁, u_{N-K₂+1}, …, u_N]
  – That is, first the K1 smallest eigenvectors, then the K2 largest ones.
  – Likewise, combine the corresponding eigenvalues in the same order.

     - node_features: the feature matrix X
     - labels: the vector (or array) Y representing node classes

• Package all these variables into an instance of GraphData.

• Return this GraphData object so that it can be used by subsequent modules (such as the trainer and evaluation classes).
