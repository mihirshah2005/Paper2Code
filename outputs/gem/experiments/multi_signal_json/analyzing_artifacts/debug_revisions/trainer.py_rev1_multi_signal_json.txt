3. LOSS COMPUTATION AND FORWARD PASS

• The Model class (from model.py) implements a compute_loss() method that must receive as inputs:
  – The real eigenbasis (or projection matrices) and the corresponding eigenvalues (from GraphData).
  – The synthetic eigenbasis and synthetic node features held within the Model instance.
  – Additionally, the method computes the category-level representations (H and H′ for the discrimination constraint) using node features and possibly the spectrum information.
  
• Loss Terms:
  – L_e (Eigenbasis Matching Loss): Measures the Frobenius norm difference between the projection matrices (u·uᵀ) for each selected eigenvector. The formulation might be:
    L_e = Σₖ ||(uₖ uₖᵀ) − (u′ₖ u′ₖᵀ)||²_F.
  – L_d (Discrimination Constraint Loss): Uses a cosine similarity between the real category-level representations and the synthetic ones.
  – L_o (Orthogonality Regularization): Ensures U′_K remains orthogonal (e.g., ||U′ᵀ_K·U′_K − I||_F).
  – The final loss is computed as:
    L_total = α · L_e + β · L_d + γ · L_o,
  using the weights provided in config (alpha = 1.0, beta = 1.0, gamma = 1.0).

• Trainer’s Role:
  – Call model.compute_loss() each epoch.
  – The returned loss is then backpropagated.
  – This encapsulation allows the Trainer to remain unaware of the internal details of each loss term.

4. OPTIMIZER SETUP & BACKPROPAGATION

• Using the Adam optimizer (as specified in the config) is required.
  – Create one optimizer for the parameters associated with the synthetic eigenbasis (e.g., model.U_syn) using learning_rate_eigenvecs (0.01 as in the config).
  – Create a separate optimizer for the synthetic node features (e.g., model.X_syn) using learning_rate_feat (1e-05 per config).
  
• In each epoch:
  – Set gradients to zero for both optimizers.
  – Perform a forward pass to obtain the loss.
  – Backpropagate gradients (loss.backward()).

    If (current_epoch % (tau1 + tau2)) < tau1: perform optimizer_eigen.step().
    Else: perform optimizer_features.step().

• This strategy ensures that in some iterations, only the eigenbasis is updated while in others, only the node features are updated. This alternating update is key to balancing the influence of eigenbasis matching versus the discrimination constraint update.
