2. DatasetLoader.load_data – Data Loading and Splitting

• This method is responsible for loading the raw graph data:  
  - It reads the graph from files or online sources based on config.dataset.name (for example, “Pubmed”).  
  - It then extracts the node features (X), node labels (Y), and constructs the full adjacency matrix (A) using NetworkX (or a similar library).

  - For the “public” split, the loader will attempt to load a predefined set of indices for training, validation, and test partitions (if such a split file is available).

For example, if the paper specifies exact ratios for train/validation/test (or for heterophilic datasets, using 2.5%/2.5%/95%), these ratios must be used to generate uniform random splits while logging that a fallback is being used.  
  - Ultimately, the method should store or return these split indices along with the GraphData so that downstream modules have consistent partitioning for evaluation.

3. DatasetLoader.truncated_eigen_decomposition – Merging, Ordering, and Robust Error Handling

• The core objective here is to compute a truncated eigen-decomposition of the normalized Laplacian (L) to obtain two groups of eigenpairs:  
  - K₁ smallest (low-frequency) eigenpairs and K₂ largest (high-frequency) eigenpairs, where K₁ + K₂ = K (and K is determined based on the synthetic node count computed from the compression ratio).  
• To ensure correct ordering in the final merged result (as required by the paper: U_K = [u₁, …, u_K₁, u_{N-K₂+1}, …, u_N]):  
  - The loader will first call scipy.sparse.linalg.eigsh with parameters to compute the K₁ smallest eigenpairs (using mode such as WHICH='SM').  
  - It will explicitly sort these eigenpairs in ascending order by their eigenvalues.  
  - Next, it will independently compute the K₂ largest eigenpairs (using WHICH='LA').  
  - These K₂ eigenpairs will be sorted as well (typically in descending order by eigenvalue if returned directly) and then reversed if necessary so that, when concatenated after the smallest group, they appear in increasing order consistent with the paper’s definition.  
  - The final result is a single, concatenated, and strictly ordered array of eigenvalues and a matching eigenvector matrix with columns corresponding to the eigenpairs.
• Robust error handling is implemented via try‑except blocks around the eigsh calls.  
  - In case of a convergence error or numerical issues, the loader will attempt a fallback: for instance, adjusting the solver’s tolerance or increasing the maximum iterations.  
  - If the fallback still fails, an explicit exception is raised with detailed logging of the encountered error.  
  - This procedure is essential for reproducibility, particularly when processing large or ill-conditioned Laplacian matrices.
