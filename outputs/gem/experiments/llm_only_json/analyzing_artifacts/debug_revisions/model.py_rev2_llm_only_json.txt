3. Loss Computation ( compute_loss )
──────────────────────────────
The compute_loss function calculates three loss components:
 A. Eigenbasis Matching Loss (Lₑ)
 B. Discrimination Constraint Loss (L_d)
 C. Orthogonality Regularization Loss (Lₒ)

A. Eigenbasis Matching Loss (Lₑ):
 • Goal: Align each column of the synthetic eigenbasis U_synth with the corresponding column from the target real eigenbasis.
 • Implementation Choice & Clarification:
  – One approach is to compute the cosine similarity for each column between U_synth and the provided target real eigenbasis (assumed preprocessed to match the synthetic graph’s dimensions).  
  – To address sign ambiguity, take the absolute value of the cosine similarity:
   Loss_k = 1 – |cosine(u_real[:, k], U_synth[:, k])|
  – Lₑ is then computed as the average (or sum) over all K columns.
 • Note: While this column‑wise cosine similarity formulation is reasonable and addresses sign ambiguity, it represents one possible metric. If the original paper intended a metric based on comparing projection matrices (e.g., ||u_real u_realᵀ − U_synth U_synthᵀ||_F), further justification and reference should be added. Document your choice clearly.
 
B. Discrimination Constraint Loss (L_d):
 • Goal: Enforce that the synthetic graph preserves the same category‐level (class) information as the real graph.
 • Expected Inputs:
  – H: the real graph category-level representation.
  – H′: the synthetic graph category-level representation.
  – Both H and H′ are expected to be externally computed matrices (e.g., with shape (C, d), where C is the number of classes and d is the feature dimension). They should be normalized appropriately (e.g., row‑normalized) so that cosine similarity is meaningful.
 • Implementation:
  – Compute cosine similarity between H and H′ along the feature dimension. For instance, it is common to compute similarity per category (i.e., row-wise) and then average the values.
  – One formulation is:
   L_d = 1 – mean(|cosine_similarity(H[i], H′[i])| for all categories i)
 • Clarification: Clearly document that H and H′ are expected to be of the same shape and that normalization must be handled before computing the loss, ensuring consistency across implementations.
 
C. Orthogonality Regularization Loss (Lₒ):
 • Goal: Ensure that the synthetic eigenbasis remains (approximately) orthonormal.
 • Implementation:
  – Compute the Gram matrix: G = U_synthᵀ · U_synth.
  – Use the Frobenius norm to measure deviation from the identity: Lₒ = ||G − I_K||_F², where I_K is a K×K identity matrix.
 
Finally, the total loss is:
 L_total = α · Lₑ + β · L_d + γ · Lₒ
 with weights α, β, and γ drawn from the configuration (from config.yaml, each is 1.0 in this case).

4. Parameter Update Methods ( update_eigenbasis and update_features )
──────────────────────────────
• These methods are provided as explicit alternatives to using standard optimizers.
• Critical Aspect:
 – To avoid interfering with PyTorch’s autograd, any manual update must be done inside a "with torch.no_grad():" block.
 – For example, in update_eigenbasis, instead of a direct subtraction on parameters, wrap:
  with torch.no_grad():
   self.synthetic_eigenbasis -= lr * gradients
 – Similarly for update_features.
• Note: In conventional use, an external optimizer (e.g., torch.optim.Adam) is preferred, so these functions should be used only if explicit manual updates are required.

5. Graph Reconstruction ( reconstruct_graph )
──────────────────────────────
• Purpose: Reconstruct the synthetic graph’s Laplacian and adjacency matrix using the learned synthetic eigenbasis and the real graph spectrum.
• Inputs:
 – real_eigenvalues: a torch.Tensor of shape (K,) containing the selected real eigenvalues corresponding to the target eigenbasis.
• Implementation:
 1. Normalize U_synth (as in forward()).
 2. Build two diagonal matrices:
  – D₁ = diag(1 – λₖ) for k = 1,...,K (for reconstructing the synthetic adjacency matrix A′).
  – D₂ = diag(λₖ) for k = 1,...,K (for reconstructing the synthetic Laplacian L′).
 3. Compute: 
  – A′ = U_synth · D₁ · U_synthᵀ
  – L′ = U_synth · D₂ · U_synthᵀ
• Return the tuple (A′, L′).
