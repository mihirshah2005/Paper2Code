3. Loss Computation – compute_loss Method

This method computes the composite loss used to train the synthetic graph model. The loss is a weighted sum of three components: eigenbasis matching loss (Lₑ), discrimination constraint loss (L_d), and orthogonality regularization loss (L_o).

A. Eigenbasis Matching Loss (Lₑ)
 • Purpose: To align the learned synthetic eigenbasis U′_K with the corresponding eigenbasis (U_real) extracted from the real graph.
 • Implementation Details:
  – Compute the projection matrices for both eigenbases (i.e., U_real @ U_realᵀ and U′_K @ (U′_K)ᵀ).
  – Calculate the Frobenius norm of their difference:  
   Lₑ = || (U_real U_realᵀ) – (U′_K (U′_K)ᵀ) ||²_F.
 • Note: It is assumed that U_real is preprocessed (e.g., truncated or projected) so that its dimensions are compatible with U′_K.

B. Discrimination Constraint Loss (L_d)
 • Purpose: To preserve category-level (class) information such that the synthetic graph exhibits similar inter-class and intra-class behavior as the real graph.
 • How Representations Are Derived:
  – For the real graph, category-level representations (H) are assumed to be provided or computed externally (e.g., by aggregating the node features X of the real graph according to their class labels).
  – For the synthetic graph, the synthetic category-level representations (H′) are derived directly from the learned synthetic node features X′.
   • One common approach is to assume that synthetic node labels Y′ are available (pre-assigned to mirror the real graph’s label distribution).
   • Then, for each unique class label c, H′_c is computed as the mean (or weighted average) of the synthetic features corresponding to nodes with label c:
    H′_c = mean { x′_i for all i with Y′_i == c }.
  – These computed H′ representations are then compared with H using cosine similarity.
 • Implementation Details:
  – Compute cosine similarity between each corresponding category representation in H and H′.
  – The loss can be expressed as:  
   L_d = 1 – mean( cosine_similarity(H, H′) ),  
  which encourages high similarity between real and synthetic category-level features.
 • This explicit derivation of H′ from X′ addresses the critique and clarifies that the synthetic node features are aggregated by their labels to compute class prototypes for enforcing the discrimination constraint.

C. Orthogonality Regularization Loss (L_o)
 • Purpose: To ensure that the learned synthetic eigenbasis U′_K remains as close as possible to an orthonormal set.
 • Implementation Details:
  – Compute the Gram matrix G = (U′_K)ᵀ @ U′_K.
  – Enforce that G is close to the identity matrix by computing:  
   L_o = || G – I ||²_F,
 where I is the identity matrix of size K×K.
 • This regularization helps prevent degenerate solutions during optimization.

D. Total Loss Combination

 • The compute_loss() method returns L_total, which is then used by the Trainer module for backpropagation and optimization.
