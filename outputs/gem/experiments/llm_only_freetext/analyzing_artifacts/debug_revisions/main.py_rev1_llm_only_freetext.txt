   • The configuration contains separate groups for training (epochs, learning rates for node features and eigenvectors, tau1/tau2, loss weights α, β, γ, and number of runs), synthetic graph settings (compression_ratio, r_k), evaluation settings (GNN architecture parameters for spatial and spectral models), and dataset info (name and split method).

2. Instantiate the Dataset Loader and Load Data

   • Call the load_data() method to obtain a GraphData object.  
   • GraphData should include:  
  – The real graph’s adjacency matrix (A)  
  – Node features (X)  
  – Labels (Y)  
  – The normalized Laplacian (L)  
  – Its truncated eigen-decomposition (selected eigenvalues and eigenvectors).  
   • Check that the dataset (e.g., Pubmed) is loaded correctly, and log key statistics (number of nodes, features, etc.).

3. Initialize the Model  
   • Instantiate the Model class, passing in a parameters dictionary that includes the training and synthetic graph settings (loss weights, learning rates, compression ratio, and r_k).  
   • Inside the Model, synthetic node features (X′) are initialized – either with a random initialization or using a pre-trained MLP as described.  
   • The synthetic eigenbasis (U′_K) is also initialized here using either a stochastic block model strategy (if available) or a random orthogonal initialization.  
   • The model encapsulates the forward() method (computes synthetic graph properties), the loss computation (for eigenbasis matching loss L_e, discrimination loss L_d, and orthogonality regularization L_o), and a method reconstruct_graph() that combines learned U′_K with the real spectrum to produce the synthetic Laplacian and adjacency.

4. Setup the Trainer and Run the Training Loop  
   • Instantiate the Trainer class with the Model object, GraphData, and the full configuration.  
   • The Trainer is responsible for the training loop:
  – The loop runs for the number of epochs specified (1500 in this case).  
  – It implements an alternating update schedule based on tau1 and tau2 (here tau1 is 0, and tau2 is 5). For example, in iterations where t % (tau1 + tau2) is less than tau1, the trainer updates the synthetic eigenbasis; otherwise, it updates the synthetic node features.  
  – At each iteration, the Model’s forward() is called to compute intermediate predictions and losses.  
  – The aggregated loss L_total (αL_e + βL_d + γL_o) is computed, and backpropagation is used to update parameters using the Adam optimizer with the graph feature learning rate and eigenvector learning rate from the configuration.  
   • During training, the Trainer also logs training losses and intermediate statistics (e.g., total variation metrics for the synthetic graph) for monitoring and debugging.

5. Reconstruct the Synthetic Graph  
   • After the training loop completes, the Model’s reconstruct_graph() method is called.  
   • This method uses the learned synthetic eigenbasis together with the real graph’s eigenvalues to compute:
  – The synthetic Laplacian, L′ = Σₖ λ_k (u′_k u′_kᵀ)  
  – The synthetic adjacency, A′ = Σₖ (1 – λ_k) (u′_k u′_kᵀ)  
   • The output (A′, X′) represents the distilled graph that will be used for evaluation.

6. Evaluate the Distilled Graph
